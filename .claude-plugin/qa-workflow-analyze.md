---
name: qa-workflow analyze
description: Analyze requirements and identify test scenarios
type: command
---

Analyze requirements to identify test scenarios with the following steps:

1. **Requirement Analysis**
   - Review requirement documents
   - Identify user stories and acceptance criteria
   - Extract testable requirements

2. **Scenario Identification**
   - Break down requirements into test scenarios
   - Identify edge cases and boundary conditions
   - Document assumptions and dependencies

3. **Test Strategy**
   - Determine testing approach
   - Identify risk areas
   - Plan coverage strategy

## Outputs
- Scenario analysis document
- Test strategy document
- Identified risks and dependencies
