---
name: qa-workflow execute
description: Execute tests and track results
type: command
---

Execute test cases and track test results with the following steps:

1. **Test Execution**
   - Execute approved test cases
   - Follow test steps precisely
   - Record test results (Pass/Fail/Skip)
   - Document actual results

2. **Issue Tracking**
   - Identify test failures
   - Document failure details
   - Create bug tickets (if needed)
   - Link failures to requirements

3. **Progress Tracking**
   - Track execution progress
   - Maintain test execution log
   - Update test status

## Outputs
- Test execution log
- Test results summary
- Bug/defect tickets (if any)
- Test execution metrics
