---
name: qa-workflow review
description: Quality review of test cases (QA Lead gate)
type: command
---

Perform quality review of test cases as a QA Lead gate with the following steps:

1. **Coverage Review**
   - Verify coverage against requirements
   - Identify gaps in test scenarios
   - Check edge case coverage

2. **Quality Validation**
   - Verify test case clarity
   - Check test step precision
   - Validate expected results

3. **Approval Decision**
   - Approve test cases
   - Request modifications (if needed)
   - Document review findings

## Outputs
- Review report with findings
- Approval/rejection decision
- List of required modifications (if any)
- Sign-off documentation
